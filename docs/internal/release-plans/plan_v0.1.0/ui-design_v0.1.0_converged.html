Primary design

2 · Tag list view

Row-based and click-sortable for fast scanning, not a drag-drop canvas. Multi-select with a select-all that distinguishes "this page" from "all matches" when paginated, and a bulk-actions bar. Built to stay responsive at thousands of tags via windowed rendering. The "Rule" column links to the effective rule - a tag matched by several rules shows its highest-priority winner (after the Q-005 fix), with the rest one click away. The same state banner from section 1 docks at the top of this view whenever Preview mode is on or Tag Curator is off, so the state is never hidden.

3 · Rule editor (card view + right-docked preview)

Round-3 pin 12 asked for a card view with a right-docked preview instead of a left-docked master pane; pin 17 asked to drop the formal wizard and use the same IA for create + edit. Both addressed. The main area shows rules as cards (name, type, enabled toggle, "N tags affected" chip). Clicking a card swaps the main area to edit mode; clicking the "New rule" card opens edit mode with sensible defaults - no separate wizard. A right-docked scrollable preview stays visible across both views: vault-wide affected tags in card view, filtered to the selected rule in edit mode. Per D-009, priority is hidden from the UI (custom rules default to 50; presets keep their values). v0.2 lights up drag-to-reorder (B012).

Primary design - redesigned from round-2 review

4 · First-run welcome modal

Fires once on first enable. Round-2 feedback drove the rebuild: the buttons used to imply the plugin wasn't enabled yet (it is), the contract cards were centered chunks that read awkwardly, integrations were wrapped-paragraph rows with disjointed alignment, and the two-presets section was buried as a one-liner. New structure: acknowledges current state in the header, makes the safety promises a single left-aligned check group, surfaces the two default presets as toggleable cards (so the user can change them before committing), and renders integrations as per-plugin cards with name + state pill + bulleted "what changes" items. Footer offers a clear binary: start curating, or start in preview mode - with a one-sentence explainer so preview mode is not a mystery.
