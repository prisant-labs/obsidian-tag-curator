Aliases & display-merge

Solves the "I have #ai, #AI, #Ai, #artificial-intelligence" problem without modifying any note content. Declare a canonical tag and its aliases; Tag Curator presents them as a single row across every surface. File rewrites stay delegated to Tag Wrangler.

Draft v0.2 target Display-only (D-004) Reuses TagMeta.aliases
Primary design

1 - Settings > Aliases tab (card view)

Replaces the v0.1 deferred placeholder. Same card-and-edit-mode pattern as the rule editor (D-010): each alias group is a full-width card showing canonical, alias count, total instances, and an enable toggle. Click any card to enter edit mode. A toolbar link to Suggested merges handles the curation side.

2 - Edit mode

Mirrors the rule editor edit pattern: breadcrumb row + prominent title + sectioned form (Canonical / Aliases / Preview). Right-docked preview becomes per-group: stats + the list of merged tags. Send-to-Tag-Wrangler offers the file-rename escape hatch.

3 - What the user sees in the native tag pane

When the group is on, alias rows are hidden via a CSS class and the canonical row gets a "+3" badge alongside its count. Clicking the badge expands the row inline (one click in, one click out) so users can audit without leaving the pane.

Tag list integration

4 - Tag list view rows fold

When enableAliases is on, the Tag list view (sidebar leaf + Settings tab per D-011) uses the folded display map. Each canonical row gets a "+N aliases" affordance that expands inline. Sort by Count uses the folded count.

Curation

5 - Suggested merges panel

Detects tag pairs that look like intended duplicates (Levenshtein <= 1, plus a case-only fast path). Threshold: combined count >= 5. Pairs the user has dismissed are persisted in ignoredMergeSuggestions. Clicking Merge pre-fills the alias editor; Ignore hides the pair from future runs.
