Scope expansion: graph + autocomplete + properties

v0.1 ships tag filtering in the native tag pane only. v0.2 extends Tag Curator to every surface where Obsidian renders tags: the graph view, the autocomplete dropdown, and the properties chip area. Same engine, same rule model, three new observers.

2 - Graph view

Hidden tag nodes disappear via display: none on a CSS class; connected edges fade to 5% opacity so the layout doesn't shift. Flagged nodes (Preview mode) gain a warning-colored ring.

3 - Editor autocomplete dropdown

When the user types # in the editor, Obsidian's tag suggestion dropdown opens. Tag Curator filters items matching hidden tags by adding the tag-curator-suggestion-hidden class. Keyboard arrows skip them naturally because display: none removes them from the flow.

4 - Properties chips

When a note has tags: in its frontmatter, Obsidian renders each tag as a chip. Tag Curator hides chips matching hidden tags - same display-only contract as everywhere else. The underlying frontmatter is untouched.

5 - Default scopes for new rules

A small multi-toggle in the General panel that pre-selects which scopes are enabled when the user creates a new rule. v0.1's defaults stay (tag-pane only); users opt new rules into broader scopes by checking these once.

Settings - Tag Curator - General
Default scopes for new rules
When you create a new custom rule, these scopes are pre-checked in the Rule editor's Scope row. Changes here do not affect existing rules.
