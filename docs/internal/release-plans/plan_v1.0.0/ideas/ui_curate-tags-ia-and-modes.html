Tag Curator · v1.0.0 · design exploration (rev 4)

Two surfaces, two jobs

A first-class Curate Tags settings tab replaces the "Open Tag Curator" button (more discoverable, and the hidden-pane bug disappears). Because Settings is a deliberate, sit-down context, the tab is always Manage - the full grid. The View / Manage switch lives only on the optional Tag Curator Pane, the ambient sidebar leaf you keep open while you write. Click through both below.

Surface 1 · in Settings
Curate Tags tab always Manage
  • A permanent tab right of General - you go here on purpose
  • Full grid: select, bulk actions, overrides, per-row menu
  • Density handled by the Columns picker + chips below search
  • No mode toggle - Settings is inherently a manage context
Surface 2 · opt-in sidebar leaf
Tag Curator Pane View / Manage
  • Enabled by a toggle in General - ambient, lives in the sidebar
  • View: click a tag to search its notes; glance at state
  • Manage: per-row + bulk actions and overrides (full parity)
  • The only surface that sits live beside the native tag pane

FAQ
Does Tag Curator change my notes?No. It is display-only - it hides or flags tags in the UI and never edits note content. Disabling or uninstalling restores every tag.
Where did a tag go?A preset, rule, or override is hiding it. Use "Why is this affected?" on its row, or run Panic disable to clear all effects at once.

Where each control lives

Curate Tags tab
Always Manage. Full grid: select + bulk bar, overrides, per-row menu, all columns (trim with the Columns picker), full chip set, chips below search. No view mode - you opened Settings to do work.
Tag Curator Pane - View
Navigate, do not curate. Click a tag to open an Obsidian search for every note using it (native-pane behavior). Each row also shows the curation state dot. Read-only otherwise.
Tag Curator Pane - Manage
Full parity with the Settings grid. Per-row menu AND selection + bulk bar + overrides. Just narrower and ambient - act on tags without leaving your note.
The principle
View navigates, Manage curates. One two-mode pane answers both "take me to this tag's notes" and "change how this tag behaves" without ever opening Settings.

Filter glossary + open decisions

Shown (new)
Tags currently visible - the inverse of Hidden. Your "Visible only" request. Added to both surfaces.
Hidden
Tags a rule or override is currently removing from view.
Flagged
A matched tag while Preview mode is ON - marked, not hidden, so you can preview a rule before committing. Empty unless Preview mode is on.
Orphans
Tags used in one note or fewer - cleanup candidates.
Frontmatter / Inline
Tags coming ONLY from the note's tags: property, or ONLY from inline #tags in the body. ("both" = appears in each.)
Unreviewed
Decision: build it for v1.0. A "Mark reviewed" action (per-row + bulk) plus a persisted reviewed flag turns this into a real triage inbox - new tags land here until you clear them.
