The Curation Workspace, docked beside the tag pane

The active curation loop leaves the Settings modal and moves into a workspace leaf that lives beside the native tag pane. You edit a rule on the left and watch every tag hide, flag, or return on the right, in real time, without a single context switch. This is the whole UX win: see, edit, preview, act, in one continuous glance.

Edit a rule in the Curation Workspace (left). Watch #FFAA00 and #section-3 strike through in the native tag pane (right), live. No reopening Settings, no guessing. One continuous glance: edit here updates there, instantly. (D-013)

Display-only and file-safe: striking a tag through changes only what the tag pane renders, never note content. Every effect is reversible by disabling the rule, the scope, or the plugin. The Curation Workspace, the four scopes (tag pane, Notebook Navigator, Properties, Autocomplete - D-014), the per-tag overrides (D-015), and the inline editor (D-010) are all v1.0 "Curation, in context". Notebook Navigator coupling is runtime-interop only; Tag Curator is Apache-2.0 and never copies NN source.
