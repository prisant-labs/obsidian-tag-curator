Curation Workspace, docked beside the tag pane

What this is. The hero of v1.0 (D-012/D-013). Why a leaf instead of Settings? A full-screen Settings modal covers the very tag pane you are curating, so you can never watch your edits land. The curation loop instead lives in a workspace leaf docked beside the native tag pane: edit a rule on the left, watch tags hide / flag / return on the right in the same glance. Contains the stats toolbar, filter chips, bulk-action bar, the virtualized tag table, the "why is this hidden?" diagnostic, per-tag override pins (D-015), and the inline card rule editor with a right-docked live preview (D-010).

What to check. Is the split the right default? Are the bulk-bar verbs and filter chips the right set and order? Does "why is this hidden?" read clearly? Are the override badges (always show / always hide) legible against the rows?

6 · Secondary navigation: section anchors vs sub-tabs (your NN example)

What this is. Reacting to the Notebook Navigator pattern you shared (a primary tab row plus a lighter secondary row). There are two ways to build that second row, and which to use depends on how heavy the section is.

Anchors (recommended for light sections): the secondary row is jump-to-section links that scroll one page of setHeading sections - closest to Obsidian's native convention, everything stays discoverable by scrolling, and it scales by simply listing each section. Sub-tab views (separate panels) only earn their keep when a section is a heavy interactive surface you would not want loaded or scrolled inline - for us that is Custom rules, which opens the full editor + live preview.

Scaling rule: use anchors for light, scroll-friendly sections (Scopes, Integrations, General); use a real sub-tab only where a section is a standalone heavy surface - for Tag Curator that is Rules, where "Presets" and "Custom rules" each deserve their own panel because Custom rules opens the editor. The anchor bar gives you NN's wayfinding without that maintenance cost. Primary tabs can still carry lucide icons (via setIcon), NN-style, if you want the visual. GeneralScopes & integrationsRules 6AdvancedHelp

A scope is a place in Obsidian where your tags appear - the tag pane, a note's Properties, the autocomplete popup, or a plugin's tag tree. Tag Curator can hide or flag tags in each:

  • Each scope is its own on/off switch.
  • Toggling one takes effect immediately - no restart.
  • Turning it off restores those tags; your notes never change.

7

Panic disable: everything off, everything back

What this is. The safety hatch (command "Panic disable"). It disables every observer, sweeps all four tc-* namespaces off the whole document, persists enabled = false, and shows a Notice. Every previously hidden row / pill / suggestion returns; the banner flips to Off and the status bar reads Tag Curator: off. Fully reversible; nothing in notes ever changed.

What to check. Is the Notice copy reassuring? After panic, are all four surfaces truly back to normal (the live smoke test is where you confirm this)?
